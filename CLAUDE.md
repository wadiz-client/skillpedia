## 작업 지침

- 한국어와 존댓말로 작성합니다.
- 능동형 문장으로 작성합니다.
- 표준 맞춤법과 띄어쓰기를 준수합니다.
- 추정으로 단정하지 않습니다.
- 이모지를 사용하지 않습니다.

## 탐색 지침

- Glob, Grep 도구를 우선 사용합니다.
- 전체 파일 읽기 전 관련성을 먼저 확인합니다.
- 다음에 해당하는 경우에만 작업 목적에 맞는 Agent 도구를 사용합니다:
  - 검색어가 모호한 경우
  - 여러 파일 탐색이 필요한 경우
  - 3회 이상 검색이 예상되는 경우

## 코드 지침

- 주석은 한국어와 존댓말로 주석이 필요한 코드에만 간결하게 작성합니다.
- 명시적 요청이 없는 경우 기존 코드 패턴을 유지합니다.
- 조건문은 긍정문을 먼저 작성합니다.
- 단일 if문에도 중괄호 {}를 사용합니다.
- 화살표 함수는 단일 표현식이라도 블록 본문과 명시적 return을 사용합니다.
- 타입을 import하는 경우 별도 행에 `import type`으로 작성합니다.
- React 내장 타입은 전역 `React` 네임스페이스로 참조합니다. (예: `React.ChangeEventHandler`, `React.FormEventHandler`, `React.ReactNode`)
- 컴포넌트는 선언(상태·라우터 훅) → 메서드 → 핸들러 → 부수 효과 훅(`useEffect`) → 렌더 순으로 작성합니다. 부수 효과 훅이 의존하는 함수보다 아래에 위치하도록 합니다.
- 코드 분석 및 작성이 필요한 경우 다음 문서를 참조합니다:
  - 프로젝트 정보: [README.md](./README.md)
  - 용어 사전: [GLOSSARY.md](./GLOSSARY.md)
  - 네이밍 컨벤션: [NAMING_CONVENTIONS.md](./NAMING_CONVENTIONS.md)

## 커밋 메시지 지침

- 한국어로 작성합니다.
- 스테이지 변경 사항만 작성합니다.
- 스테이지 변경 사항이 없는 경우 전체 변경 사항에 대해 작성합니다.
- 형식: `type: 제목`
- 타입(`type`)은 다음 중 하나를 사용합니다:
  - feat: 기능 추가 및 수정
  - fix: 버그 수정
  - docs: 문서 작성
  - style: 코드 포맷팅
  - refactor: 리팩토링
  - test: 테스트
  - chore: 환경 설정
- 제목은 변경 사항을 명령형으로 작성합니다.
- 본문은 생략합니다.
- 본문은 변경 사항이 매우 복잡한 경우에만 내용과 이유를 1레벨 목록형으로 작성합니다.
- 커밋 메시지에 출처 표기 문구는 사용하지 않습니다.

## 토큰 최적화 지침

- 답변은 가능한 한 짧게 작성합니다.
- 불필요한 서론·결론, 사족, 동일 의미의 표현을 금지합니다.
- 변경 사항은 Edit 도구로 수정 블록만 제공합니다.
- 전체 파일을 출력하지 않습니다.
- 요구 사항이 모호한 경우 출력 전에 반드시 확인 질문을 합니다.
- 독립적인 도구 호출은 병렬로 실행합니다.
- 가능한 경우 응답 말미에 현재 메시지 기준으로 토큰 사용량을 보고합니다.
  - 시스템 메시지의 토큰 값은 누적값입니다. 직전 메시지와의 차이를 계산합니다.
  - 천 단위 구분 기호를 사용합니다.

  ```text
  - 입력 토큰: N
  - 출력 토큰: N
  - 남은 토큰: N
  ```
